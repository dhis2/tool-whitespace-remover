Help and instructions for Whitespace Cleaner!

It is crucial to first run this tool in a development or test environment to ensure that there are no disruptions to your production environment.

Key Features

  1. Check and Fix Buttons: Each metadata entry has individual “Check” and “Fix” buttons.
  2. Batch Actions: Selected entries can be checked and fixed in batches using the “Check Selected” and “Fix Selected” buttons.
  3. Conflict Identification: Highlights conflicts such as duplicate names, short names, or codes.
  4. User Feedback: Provides visual status updates on the readiness and conflicts of each entry.

Getting Started

  1. Launch the Tool: Open the Whitespace Cleaner Tool in your web browser.
  2. Environment Setup: Always start by working in a development or test environment.
  3. Initial Data Fetch: The tool will automatically fetch metadata types (e.g., name, code, description) and display them in separate tabs.

Using the Tool

Tabs and Navigation

  • Tabs: Each metadata type is represented by a tab.
  • Active Tab: Click on a tab to view its contents. The active tab is highlighted.

Viewing Metadata

  1. Select Tab: Click on the tab corresponding to the metadata type you want to view.
  2. Table View: Metadata entries are displayed in a table format with columns for ID, Name, Short Name, Code, and Description.
  3. Selecting Rows: Use the checkboxes to select specific rows for batch actions.

Individual Actions

  1. Check Individual Entries:
    • Click the “Check” button next to an entry to check for conflicts.
    • The status cell will update to show if the entry is ready or if there are conflicts.
  2. Fix Individual Entries:
    • If the status is “Ready,” click the “Fix” button to clean the metadata.

Batch Actions

  1. Check Selected Entries:
    • Select rows using the checkboxes.
    • Click the “Check Selected” button to check all selected rows for conflicts.
    • The status cells will update accordingly.
  2. Fix Selected Entries:
    • Ensure all selected rows are marked as “Ready.”
    • Click the “Fix Selected” button to clean metadata for all ready entries.

Status Indicators

  • Conflicts: If conflicts are found, details will be displayed, and the status will be marked as “Conflict.”
  • Ready: If no conflicts are found, the status will be marked as “Ready.”

Handling Empty Tables

  • When all entries of a type are fixed and the table is empty, the corresponding tab will be removed.
  • Focus will shift to the previous tab.
  • If no tabs remain, a “Success!” message will be displayed.

Best Practices

  1. Development or Test Environment: Always begin by running the tool in a development or test environment.
  2. Review Conflicts: Carefully review and resolve conflicts before fixing multiple entries.
  3. Batch Processing: Utilize batch actions for efficiency but verify readiness status before applying fixes.

Troubleshooting

  • Conflicts Found:
    • If conflicts are detected, resolve duplicate names, short names, or codes before proceeding.
  • Errors During Fixing: Review error messages and ensure entries are marked as “Ready” before retrying fixes.
